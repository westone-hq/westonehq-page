매 학기 반복되는 두 가지 작업이 있었습니다.

성적 처리

LMS에서 내려받은 진도·과제 엑셀을, 14주 출석을 일일이 확인하고, 대학별로 나누고, 점수를 환산해 수료 여부를 판정하고, 보고서로 정리. 과목 수 × 대학 수만큼 시트가 불어나면 수작업 실수가 나기 시작합니다.

설문 분석

프로그램 전후로 받은 설문을 학생마다 짝지어, 여러 항목을 통계로 검증하고 Word 보고서로. 학기마다 수백~수천 건을 손으로 처리하는 데 며칠이 걸렸습니다.

게다가 이런 제약이 있었습니다.

  • 비전문가도 파일만 넣으면 결과가 나와야 함
  • 학생 데이터라 인터넷 없는 PC에서 처리해야 함 (보안)
  • 대학마다 분류 기준·점수 비율이 달라 설정이 바뀔 수 있어야 함

파일을 끌어다 놓으면, 끝납니다.

오프라인에서 동작하는 데스크톱 앱입니다

Windows 데스크톱 앱(Electron 기반)으로 만들었습니다. 학생 개인정보가 포함된 데이터를 다루기 때문에 외부 서버로 전송할 수 없었고, 인터넷이 연결되지 않는 PC에서도 실행돼야 했습니다. Electron으로 패키징하면 설치 파일 더블클릭으로 끝나기 때문에, 개발 배경 없는 담당자도 별도 설정 없이 바로 쓸 수 있습니다.

화면은 성적 처리 탭과 설문 분석 탭 두 가지로 구성됩니다. 대학 매핑·점수 비율·수료 기준·설문 항목 수 같은 설정은 모두 화면 안에서 바꿀 수 있고, 변경 내용은 자동으로 저장됩니다. 매 학기 기준이 달라져도 코드를 건드리지 않고 담당자가 직접 조정할 수 있도록 설계했습니다.

정확성을 어떻게 보장했나

자동화에서 가장 까다로웠던 건 예외 상황이었습니다. 파일 형식만 맞으면 알아서 되는 부분보다, 실제 운영 중 마주친 예외들을 하나씩 처리하는 과정이 정확성의 핵심이었습니다.

  • 마스킹된 학생 이름 복원 — 과제 파일에서 이름이 가려진 경우, 같은 로그인 ID를 가진 진도 파일에서 실명을 찾아 자동으로 복원합니다.
  • 여러 과목 동시 수강 — 한 학생이 여러 과목을 들으면 기록이 섞일 수 있어, 과목과 기간을 함께 묶어 정확히 구분하도록 했습니다.
  • 대학 분류 — 로그인 ID의 앞부분으로 소속 대학을 판별하며, 예외는 수동으로 지정할 수 있습니다.
  • 수료 판정 — 결석 횟수, 환산 점수, 과제·시험 제출 여부 네 가지를 모두 확인해 자동 판정합니다. 기준이 코드로 고정돼 있어 사람이 매번 판단할 때 생기던 실수가 없습니다.
  • 통계 검증 — 사전·사후 설문을 대응표본 t-검정으로 자동 분석하고 유의성을 표기합니다.

학생 개인정보는 어떻게 보호했나

프로그램이 외부와 통신하지 않도록 설계했습니다. 파일을 드래그해서 결과를 받는 전 과정이 담당자의 PC 안에서만 처리되고, 학생 데이터는 그 밖으로 나가지 않습니다.

개발과 테스트 과정에서도 실제 학생 정보를 사용하지 않았습니다. 이름과 학번을 가명·마스킹 처리한 샘플 파일만으로 모든 기능을 검증했습니다.

'파일 드래그 → 자동 처리 → 결과 다운로드' 구조는 교육 외 다른 분야 반복 업무에도 그대로 적용됩니다.

  • 기업 사내 교육팀 — 이수율 집계·수료증 발급 자동화
  • 공공기관 연수원 — 학기별 성과 보고서 자동 생성
  • 학원 프랜차이즈 — 다지점 성적 취합·비교 리포트

핵심 처리 구조가 재사용되므로, 유사한 업무는 더 짧은 기간·비용으로 구축할 수 있습니다.
